# ADC Capture вҖ” AD9248 вҶ’ UART вҶ’ PC

This is the final and main sub-project of the acquisition system.
The FPGA generates a sampling clock for the AD9248 ADC, captures the
14-bit parallel data bus, packs each sample into a 3-byte UART frame,
and streams it to the PC where a Python script displays the signal in real time.

---

## What you will learn

* How a parallel 14-bit ADC (AD9248) interfaces with an FPGA
* How to generate a sampling clock from the FPGA
* How to capture parallel data synchronously using edge detection
* How to pack a 14-bit value into a multi-byte UART frame
* How to decode and visualize ADC samples in Python in real time

## Hardware connections

| FPGA pin | Signal | Direction | Description |
|----------|--------|-----------|-------------|
| P55 | clk_50m | Board osc. | 50 MHz system clock |
| P61 | ACL | FPGA вҶ’ ADC | Channel A sampling clock |
| P1вҖ“P17 | ADC_D[13:0] | ADC вҶ’ FPGA | 14-bit parallel data bus |
| P126 | uart_tx | FPGA вҶ’ PC | UART TX to USB-UART adapter |

> вҡ пёҸ BCL (channel B clock) is not connected. On this AliExpress module,
> channel B data lines are not routed to the connector вҖ” only channel A
> is usable.

## UART frame format

Each 14-bit ADC sample is transmitted as 3 bytes:

```
Byte 0 : 0xAB            вҶҗ sync marker (always 10101011)
Byte 1 : bits[13:6]      вҶҗ 8 MSBs
Byte 2 : 00_bits[5:0]    вҶҗ 6 LSBs (2 zero padding bits)
```

The sync marker `0xAB` allows the Python script to re-synchronize
if bytes are lost or corrupted.

Reconstruction in Python:
```python
raw = (b1 << 6) | (b2 & 0x3F)   # 14-bit value, range 0вҖ“16383
```

## Voltage conversion

The AD9248 uses **offset binary** encoding:

| Raw value | Voltage |
|-----------|---------|
| 0x0000 (0) | вҲ’Vref |
| 0x2000 (8192) | 0 V |
| 0x3FFF (16383) | +Vref |

```python
def raw_to_volt(raw, vref=5.0):
    return (raw - 8192) * vref / 8192
```

## Step by step

### Prerequisites вҖ” Xilinx ISE 14.7

ISE 14.7 is the last version of the Xilinx ISE toolchain (2013).
It does not run reliably on Windows 10/11 or recent Linux distributions.
**The recommended approach is to run ISE inside a Virtual Machine.**

**Recommended setup:**
- **VM software:** VirtualBox (free) or VMware
- **Guest OS:** Ubuntu 14.04 LTS or CentOS 6 (both officially supported by ISE)
- **ISE edition:** WebPACK (free, no license required for Spartan-6 XC6SLX9)

**Download ISE 14.7:**
https://www.xilinx.com/support/download/index.html/content/xilinx/en/downloadNav/vivado-design-tools/archive-ise.html

> вҡ пёҸ The installer is large (~6 GB). During installation select **WebPACK**
> and make sure **Spartan6** is checked in the device list.

> вҡ пёҸ **USB passthrough:** To program the FPGA from inside the VM, pass
> through the JTAG USB device to the guest OS.
> In VirtualBox: *Devices вҶ’ USB вҶ’ select the JTAG adapter*
> In VMware: the USB device appears automatically when plugged in.

> вҡ пёҸ **CH340 driver (Windows host):** The USB-UART adapter uses a CH340 chip.
> Install the driver on the **host** machine so Python can access the serial port:
> https://www.wch-ic.com/downloads/CH341SER_EXE.html

### 1 вҖ” Wire the hardware

Connect the AD9248 module to the FPGA board following the pin table above.
Make sure GND is shared between both boards.

Apply a test signal to the ADC channel A input:
- Function generator: 1 kHz sine wave, amplitude within ВұVref
- Or a simple DC voltage to verify the offset binary encoding

### 2 вҖ” Create an ISE project (inside the VM)

1. Open **Xilinx ISE 14.7** вҶ’ *File вҶ’ New Project*
2. Device settings:

| Field | Value |
|-------|-------|
| Family | Spartan6 |
| Device | XC6SLX9 |
| Package | TQG144 |
| Speed | -2 |

3. Add `adc_fpga.v`, `uart_tx.v`, `adc_fpga.ucf`
4. Set `adc_fpga` as the top module

### 3 вҖ” Synthesize and implement

Right-click `adc_fpga` вҶ’ *Run All Implementation Steps*

Wait for green ticks on all steps:
- вң“ Synthesize
- вң“ Translate
- вң“ Map
- вң“ Place & Route
- вң“ Generate Programming File

### 4 вҖ” Program via iMPACT

1. *Boundary Scan вҶ’ Initialize chain*
2. Right-click the device вҶ’ *Assign New Configuration File* вҶ’ `adc_fpga.bit`
3. Right-click вҶ’ *Program*

### 5 вҖ” Run the Python visualization

On the **host** machine (not inside the VM):

```bash
pip install pyserial matplotlib numpy
python ../../../python/uart_adc_reception.py
```

The script auto-detects the serial port. Two live plots appear:
- **Raw counts** вҖ” values between 0 and 16383
- **Voltage** вҖ” converted values in volts (ВұVref)

## Troubleshooting

**No data in Python / blank plot**
вҶ’ Check the CH340 driver is installed on the host machine
вҶ’ Confirm UART TX pin idles at 3.3 V with a multimeter
вҶ’ Make sure baud rate matches: `BAUD_RATE` in firmware = `BAUD` in Python (both 115200)

**Noisy or wrong values**
вҶ’ Check GND connection between FPGA board and ADC module
вҶ’ Verify ADC input signal is within ВұVref range
вҶ’ Double-check pin mapping in `adc_fpga.ucf`

**Plot shows flat line at mid-scale (8192)**
вҶ’ ADC input is floating вҖ” connect a signal or tie to GND to verify
вҶ’ Check the ACL clock is reaching the ADC (probe pin P61 with oscilloscope)

**ISE cannot find uart_tx module**
вҶ’ Make sure `uart_tx.v` is added to the ISE project
(*Project вҶ’ Add Source*)

## Note on ISE synthesis warnings

During synthesis ISE may report warnings about unconnected or constant
FF/Latches in `uart_tx`. These are expected and harmless вҖ” they appear
because some bits of the UART shift register are optimized away when
the synthesizer can predict their value. The circuit functions correctly.
